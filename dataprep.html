Data Preparation & Exploratory Data Analysis

This section describes the datasets used in this project, how they were collected, and the steps used to clean and prepare the data for analysis. The goal of this stage is to build a structured biomedical research dataset that can be used to analyze publication trends, collaboration patterns, and scientific impact over time.

1. Data Sources

The primary dataset used in this project was collected from the Europe PMC API, a large-scale biomedical literature database that aggregates research articles from PubMed and other life science repositories.

  • Source: Europe PMC (European Bioinformatics Institute)
  • Access Method: REST API
  • Query Used: "biomedical"
  • Link: https://www.ebi.ac.uk/europepmc/
  • Format: JSON (converted to tabular format in R)

In addition to the API data, future versions of this project may incorporate external datasets (such as Kaggle biomedical datasets or NIH datasets) to expand coverage and enable cross-dataset comparisons.

2. API Data Collection

Data was collected using the Europe PMC REST API. The API allows direct programmatic access to biomedical publication metadata, including titles, authors, journals, publication years, citation counts, and open access status.

The following endpoint was used: 

https://www.ebi.ac.uk/europepmc/webservices/rest/search?query=biomedical&format=json&pageSize=1000

The data was retrieved in JSON format and processed in R using the httr and jsonlite packages. The nested JSON structure was flattened into a structured data frame for analysis.

3. Raw Data Structure

The raw dataset contained approximately 1000 biomedical research papers with 33 variables per paper. Each row represents a single publication, and each column represents metadata associated with that publication.

Key variables include:

  • Title
  • Authors
  • DOI
  • Journal Name
  • Publication Year
  • Citation Count
  • Open Access Status
  • Publication Type

Figure 1: Example of raw API-derived dataset structure.

4. Data Cleaning Steps

The raw JSON data required several preprocessing steps before it could be used for analysis. Biomedical metadata is often incomplete or inconsistently structured across journals, so cleaning was necessary to ensure data quality.

Cleaning steps performed:

  • Flattened nested JSON structures into a tabular data frame
  • Converted list-based fields into character strings
  • Removed records with missing titles, DOIs, or publication years
  • Filtered dataset to include only journal articles
  • Standardized publication year format for analysis

After cleaning, the dataset was reduced from 1000 raw records to 623 clean records, ensuring higher reliability for statistical analysis and visualization.

Figure 2: Example of cleaned and structured dataset.

5. Exploratory Data Analysis (EDA)

Initial exploratory analysis was conducted to understand the structure and distribution of biomedical research publications in the dataset. This includes examining publication trends, journal frequency, and citation distributions.

Planned visualizations include:

  • Number of publications per year
  • Top 10 journals by publication count
  • Citation count distribution
  • Open access vs non-open access publications
  • Trend of publications over time
  • Author frequency analysis
  • Publication type distribution
  • Keyword or title frequency analysis
  • Collaboration size distribution
  • Correlation between citations and publication year

Figures 3–4: Example exploratory visualizations of biomedical publication trends.

6. Key Observations (Preliminary)

Early analysis suggests that biomedical research output is highly concentrated in recent years, reflecting rapid growth in scientific publishing. Journals vary significantly in publication volume, and citation counts show strong skewness, with a small number of papers receiving the majority of citations.

These patterns indicate that biomedical research behaves as a complex system, where certain topics, journals, and collaborations disproportionately influence scientific impact.

7. Summary

This dataset provides a structured foundation for analyzing biomedical research trends using data science techniques. The combination of API-based data collection and systematic cleaning ensures that the dataset is suitable for deeper analysis, including clustering, prediction, and network analysis in later stages of the project.
