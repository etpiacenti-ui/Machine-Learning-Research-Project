Data Preparation & Exploratory Data Analysis (EDA)

This section documents the systematic process by which the two biomedical literature datasets used in this project were collected, integrated, cleaned, standardized, and explored. The workflow spans multiple stages of the data science pipeline, beginning with data acquisition from an external REST API (Europe PMC) and a secondary structured dataset sourced from Kaggle. Emphasis is placed on ensuring consistency across heterogeneous data sources, particularly in harmonizing variable definitions, resolving missing values, normalizing textual fields, and addressing outliers that may affect downstream analysis.

The primary objective of this stage is to construct a robust, high-quality, and analytically coherent biomedical research dataset that is suitable for advanced machine learning applications. This includes unsupervised learning methods such as clustering and association rule mining, as well as supervised learning techniques including Naive Bayes classification, Decision Tree modeling, Support Vector Machines, regression analysis, and neural network architectures. The aim of this section is to establish a clean and well-structured dataset to ensure that subsequent modeling steps are built on a reliable foundation, thereby improving the validity, interpretability, and reproducibility of all analytical results.

1. Data Sources

This project uses three complementary biomedical literature datasets. All raw data sources and links are provided in Section 6 at the bottom of this page.

1.1 Europe PMC API Dataset

  • Source: Europe PMC (European Bioinformatics Institute)
  • Access Method: REST API
  • Purpose: Provides large-scale metadata on biomedical publications (authors, journals, indexing dates, open access status, etc.)
  • Format: JSON → flattened into CSV

API Endpoint Used:

Europe PMC RESTful Web Service Documentation 

https://www.ebi.ac.uk/europepmc/webservices/rest/search?query=PUB_YEAR:2026&format=json&pageSize=1000

This query retrieves all biomedical publications indexed in 2026. The JSON response was flattened and cleaned in R.

1.2 Kaggle Biomedical Dataset (Secondary Source)

  • Source: Kaggle public dataset
  • Purpose: Provides curated metadata such as keywords, major topics, abstract word counts, and country of origin
  • Format: CSV

The Kaggle dataset complements the API dataset by providing thematic and semantic variables not available in Europe PMC.

1.3 Semantic Scholar Papers Dataset (Third Source)

  • Source: Semantic Scholar Open Research Corpus (S2ORC)
  • Access Method: Bulk dataset download (JSONL shard)
  • Purpose: Provides large-scale metadata including titles, abstracts, authors, venues, fields of study, and open access status
  • Format: JSONL → converted to CSV

A single shard (~1.5 GB compressed) of the Semantic Scholar papers dataset was downloaded and converted from JSONL to CSV. This dataset complements the Europe PMC and Kaggle datasets by providing a broader and more diverse sample of scientific literature, including fields outside biomedicine. Only metadata fields relevant to this project were retained.

3. Data Cleaning & Standardization

Cleaning and EDA were performed iteratively. Each visualization informed the next round of preprocessing.

3.1 Cleaning Steps for Europe PMC API Data

  • Converted pubYear → numeric pub_year
  • Converted citedByCount → numeric citation_count
  • Extracted author_count from authorString using comma counts
  • Standardized isOpenAccess → Open / Closed
  • Parsed firstIndexDate and firstPublicationDate into proper date format
  • Computed lag_days = firstIndexDate - firstPublicationDate
  • Filtered out negative or impossible lag values
  • Removed missing values in key variables

3.2 Cleaning Steps for Kaggle Data

  • Converted pub_year → numeric
  • Standardized author_count from authors_count
  • Tokenized semicolon-separated keywords
  • Trimmed whitespace and removed empty tokens
  • Filtered missing major_topic and country
  • Ensured abstract_words was numeric

3.3 Cross-Dataset Harmonization

  • Standardized variable names across datasets (author_count, pub_year, journalTitle)
  • Removed duplicate column names before merging
  • Capped extreme outliers in visualizations (e.g., author_count > 50)

5. Summary

The Data Preparation process resulted in cleaned, standardized, and analytically consistent biomedical datasets. These datasets provide a multidimensional representation of biomedical research activity spanning publication trends, topical distributions, and collaborative structures for the 2026–2027 period. The resulting processed dataset serves as a validated and reproducible foundation for all subsequent machine learning workflows in this project, including both unsupervised learning (clustering and association rule mining) and supervised learning methods (classification, regression, and neural network modeling). This ensures that downstream analyses are conducted on a reliable, consistent, and methodologically sound data framework.
